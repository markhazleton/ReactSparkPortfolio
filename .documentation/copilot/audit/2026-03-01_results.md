### Detailed Security Issues

#### VALID1: No Runtime Validation Library (CRITICAL)

**Constitution Reference**: VIII. Input Validation & Security (MANDATORY)
> "Runtime validation library (Zod) MUST be used for API responses"

**Evidence**: No imports of `zod`, `yup`, or `joi` found in codebase.

**Files Affected**:
- All services (ProjectService.ts, JokeService.ts, RssService.ts)
- All components consuming external APIs

**Risk**: External API responses are not validated at runtime, which could lead to:
- Type coercion errors
- Unexpected data shapes causing crashes
- Security vulnerabilities from malformed data

**Recommendation**:
```bash
npm install zod
```

Then add validation schemas:
```typescript
// src/models/Project.tsx
import { z } from 'zod';

export const ProjectDataSchema = z.object({
  id: z.number(),
  image: z.string(),
  p: z.string(),
  d: z.string(),
  h: z.string().url()
});

export type ProjectData = z.infer<typeof ProjectDataSchema>;

// src/services/ProjectService.ts
export const fetchProjectsData = async (): Promise<ProjectData[]> => {
  const response = await fetch(projectsSourceUrl);
  const data = await response.json();
  
  // Validate runtime data
  return z.array(ProjectDataSchema).parse(data);
};
```

#### 6. **ERR1**: Implement Error Boundaries

**Constitution Violation**: VI. Error Handling & Resilience (MANDATORY)
> "React Error Boundaries MUST be implemented for component error isolation"

**Action**: Create `src/components/ErrorBoundary.tsx`:
```typescript
import React, { Component, ErrorInfo, ReactNode } from 'react';
import { Alert, Container } from 'react-bootstrap';

interface Props {
  children: ReactNode;
  fallback?: ReactNode;
}

interface State {
  hasError: boolean;
  error?: Error;
}

export class ErrorBoundary extends Component<Props, State> {
  public state: State = {
    hasError: false
  };

  public static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error };
  }

  public componentDidCatch(error: Error, errorInfo: ErrorInfo) {
    console.error('Uncaught error:', error, errorInfo);
    // TODO: Send to error tracking service
  }

  public render() {
    if (this.state.hasError) {
      return this.props.fallback || (
        <Container className="mt-5">
          <Alert variant="danger">
            <Alert.Heading>Something went wrong</Alert.Heading>
            <p>We're sorry for the inconvenience. Please try refreshing the page.</p>
          </Alert>
        </Container>
      );
    }

    return this.props.children;
  }
}
```

Wrap routes in App.tsx with ErrorBoundary.
